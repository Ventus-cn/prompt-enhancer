# Prompt Enhancer

`Prompt Enhancer` 是一个用于增强用户自然语言输入的 Codex skill。

它的作用是先将用户的原始请求整理为更清晰、更结构化、更适合 AI 执行的提示词，再基于该提示词继续执行任务或回答问题。

## 功能简介

这个 skill 主要提供三层能力：

### 1. 提示词优化

将用户输入的自然语言需求自动整理为更专业的提示词表达，通常会补齐这些维度：

- 目标
- 背景或上下文
- 约束条件
- 期望输出
- 语气或风格

### 2. 展示优化后的提示词

调用后会先展示整理后的提示词，再继续执行后续任务。

这样可以让用户明确看到：

- 当前请求被系统如何理解
- 后续执行将以什么样的任务描述为基础

### 3. 自动转调更匹配的 skill

如果当前会话中已经存在更适合该任务的 skill，`Prompt Enhancer` 会在完成提示词优化后自动转交给对应 skill 继续处理，而不要求用户手动再次调用。

## 适用场景

适合以下类型的任务：

- 写作与表达
- 分析与总结
- 策划与方案
- 编程求助
- 前端页面生成
- 文档、表格、演示文稿等结构化任务

## 工作方式

典型流程如下：

1. 接收用户原始输入
2. 判断任务意图与任务类型
3. 生成结构化提示词
4. 展示优化后的提示词
5. 检查当前会话中是否存在更匹配的 skill
6. 有则自动转调，无则由自身继续处理

## 使用方式

显式调用：

```text
$prompt-enhancer <你的需求>
```

示例：

```text
$prompt-enhancer 帮我把这段产品介绍改写得更专业一些
```

```text
$prompt-enhancer 帮我分析这段会议纪要并提炼重点
```

```text
$prompt-enhancer 帮我创建一个个人介绍网页
```

## 自动路由说明

当当前会话已经存在更匹配的 skill 时，本 skill 可以自动路由到对应能力。

常见路由方向包括：

- 前端或网页任务 -> 前端相关 skill
- 文档任务 -> 文档相关 skill
- 表格任务 -> 表格相关 skill
- 幻灯片任务 -> 演示相关 skill
- 浏览器打开、测试、导航 -> 浏览器相关 skill

自动路由仅在以下条件下发生：

- 目标 skill 已经在当前会话中可用
- 目标 skill 与当前任务明显更匹配
- 使用目标 skill 比通用增强处理更合适

## 限制说明

### 1. 不会修改输入框中的原始文本

这个 skill 不具备发送前拦截输入框并替换文本的能力。

它的工作位置是在模型执行流程中，而不是输入框编辑层。

### 2. 需要显式调用

该 skill 当前采用显式调用模式，不会默认隐式注入。

### 3. 自动路由依赖当前会话

即使某个 skill 已经安装在本地 skill 库中，只要它没有出现在当前线程的可用 skill 列表里，就不能被本 skill 自动调用。

## 文件说明

- `SKILL.md`
  Codex 实际读取的 skill 规则文件

- `READER.md`
  面向人的说明文档，适合用于 GitHub 项目介绍

- `agents/openai.yaml`
  skill 的显示信息与默认调用提示

- `references/`
  存放提示词模板与任务路由参考资料

## 当前版本特性

当前版本重点支持：

- 自然语言请求增强
- 展示优化后的提示词
- 自动路由到更匹配的可用 skill

它适合作为一个统一入口，用于降低手写提示词和手动切换 skill 的成本。
